## <Book aria-hidden="true" /> Libraries

</header>

### [`stream-download`](https://github.com/aschey/stream-download-rs)

A library for streaming audio, video, and other content. The overall goal is to
handle the intricacies involved with asynchronous streams and allow any byte
source to be used as a synchronous
[`Read`](https://doc.rust-lang.org/std/io/trait.Read.html) and
[`Seek`](https://doc.rust-lang.org/std/io/trait.Seek.html) interface.

### [`icy-metadata`](https://github.com/aschey/icy-metadata)

An [`Icecast`](https://icecast.org) metadata parser.
Icecast embeds its metadata directly within a media stream,
requiring some form of middleware to parse and remove it before the media data
can be decoded.

### [`terminput`](https://github.com/aschey/terminput)

An ANSI escape code parser/encoder for terminal inputs and
abstraction over various backends that provide input events, such as key presses and mouse clicks.

### [`termprofile`](https://github.com/aschey/termprofile)

Detects the color level supported by the current terminal using environment variables and queries.
Also supports automatically adapting colors and styles based on the detected support level.

### [`tipsy`](https://github.com/aschey/tipsy)

Cross-platform IPC for [`Tokio`](https://tokio.rs).
Uses [unix domain sockets](https://en.wikipedia.org/wiki/Unix_domain_socket) on Unix and
[named pipes](https://learn.microsoft.com/en-us/windows/win32/ipc/named-pipes) on Windows.
Many applications support Unix sockets, but a higher level of abstraction is required to make
this type of IPC paradigm work cross-platform.
